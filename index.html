Where to build first

Productivity and Project Management (10 ready today), Developer, Infra and Data platforms (9 ready today), CRM and Sales (9 ready today). These have clear docs, reusable OAuth/token patterns and predictable objects.

Where outreach matters

Ecommerce (3 blocked/unclear), Finance and Fintech (3 blocked/unclear), Data, SEO and Scraping (2 blocked/unclear). The blockers are not coding difficulty; they are account access, data licensing and public-doc gaps.

Auth predicts motion

OAuth rows are 92% buildable now/after normal gates; API-key rows are 93%. OAuth usually means app-review work, while API keys often mean faster private/internal tools.

MCP is a strategic wedge

19 apps already show official MCP or agent-native signals. That suggests Composio can prioritize wrappers around existing MCP surfaces before inventing new ones.

Agent Workflow

The pipeline uses a real first-pass research agent plus loop prompting as an operating pattern: collect evidence, classify fields, criticize likely hallucinations, then repair with human-verifiable evidence.

1. SeedStart with app names, category, and official-doc hints.
2. Search/fetchUse direct hints, web search, docs fetch and optional Composio search hooks.
3. ExtractClassify auth, access gate, API breadth, MCP and buildability with heuristics or OpenAI.
4. CritiqueFlag ambiguous rows: hidden APIs, ads review, finance compliance.
5. RepairHuman audit updates rows and records misses honestly.

Runnable Proof

Run the live first-pass research agent:

python3 src/agent.py --limit 5 --max-pages 3

Run the research artifact locally:

python3 src/research_agent.py --build

Run URL verification loop:

python3 src/research_agent.py --check-links --limit 40

Optional: OPENAI_API_KEY=... enables LLM extraction; COMPOSIO_API_KEY=... enables the Composio tool-search hook.
